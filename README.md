UI Automation Tests (Python + Playwright)

Данный проект представляет собой учебный проект по автоматизации UI-тестирования с использованием Python, Pytest,
Playwright и Allure.

Автотесты предназначены для проверки пользовательского интерфейса веб-приложения и демонстрируют базовые и продвинутые
подходы к написанию, запуску и анализу результатов UI-тестов.

⸻

Описание проекта

Цель проекта — освоить и закрепить навыки автоматизации UI-тестирования:
• написание автотестов на Python
• работа с Playwright для управления браузером
• использование Pytest в качестве тестового фреймворка
• формирование и просмотр Allure-отчетов

Проект организован по принципам чистой архитектуры автотестов и может быть расширен под реальные задачи.

⸻

Требования

Перед началом работы убедитесь, что у вас установлены:
• Python 3.9+
• Git
• Allure Commandline (для просмотра отчетов)

Проверка версии Python:

python --version

⸻

Клонирование проекта

Склонируйте репозиторий с помощью Git:

git clone https://github.com/your-username/your-repo-name.git
cd your-repo-name

⸻

Создание виртуального окружения

Рекомендуется использовать виртуальное окружение для изоляции зависимостей проекта.

Linux / MacOS

python3 -m venv venv
source venv/bin/activate

Windows

python -m venv venv
venv\Scripts\activate

После активации виртуального окружения в терминале появится префикс (venv).

⸻

Установка зависимостей

Установите все зависимости из файла requirements.txt:

pip install -r requirements.txt

⸻

Установка браузеров Playwright

При первом запуске Playwright необходимо установить браузеры:

playwright install

⸻

Запуск автотестов с генерацией Allure-отчета

Для запуска автотестов и сохранения результатов для Allure выполните команду:

pytest --alluredir=allure-results

Если в проекте используются маркеры (например, regression):

pytest -m regression --alluredir=allure-results

⸻

Просмотр Allure-отчета

После выполнения тестов откройте Allure-отчет командой:

allure serve allure-results

Отчет автоматически откроется в браузере и будет содержать:
• статус тестов
• шаги выполнения
• скриншоты (если настроены)
• логи и ошибки
